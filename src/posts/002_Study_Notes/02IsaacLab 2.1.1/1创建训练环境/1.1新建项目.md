# Isaac Lab 新建项目(linux系统)


## 第一阶段：项目初始化（Project Generation）

### 1. 进入 Isaac Lab 根目录

```bash
cd ~IsaacLab
```

> 请根据你的 Isaac Lab 实际安装路径调整。

### 2. 启动生成脚本

```bash
./isaaclab.sh --new
```

### 3. 参数填写配置

> **注意**：选择项时，先按 **空格** 选中，再按 **回车** 确认。

按照下面内容填写：

- **Template type**：`external`
- **Project name**：`UR10e_OCS_1`
- **Project path**：`/home/isst/Action/04lab_example/UR10e_OCS_1`
- **Workflow**：`Manager-based | single-agent`
- **RL library**：`skrl`
- **RL algorithms**：`all`

> **说明**：`Project path` 末尾只保留一个项目名，不要重复嵌套。

---

## 第二阶段：环境激活与项目安装（Installation）

### 1. 进入项目目录

```bash
cd /home/isst/Action/04lab_example/UR10e_OCS_1
```


### 2. 激活 Conda 环境

```bash
conda activate env_isaaclab
```

### 3. 以可编辑模式安装扩展

```bash
python -m pip install -e source/UR10e_OCS_1
```
> **注意**：UR10e_OCS_1  必须和项目名称一样。
---

## 第三阶段：核心代码修改（配置 UR10e）

需要修改的文件路径为：

```bash
source/UR10e_OCS_1/config/ur10e_catenary_inspection/env_cfg.py
```



## 第三阶段：验证运行

### 1. 检查任务列表

```bash
python scripts/list_envs.py
```

确认输出中可以看到类似如下任务名称：

```bash
Template-Ur10e-Ocs-1-v0
```

### 2. 启动可视化随机测试

```bash
python scripts/random_agent.py --task Template-Ur10e-Ocs-1-v0
```
如果看到有机器人模型加载出来，说明创建成功了。

## 第五阶段：核心开发建议

### 1. 显存控制

如果运行时报显存不足，可以在命令后添加环境数量参数，例如：

```bash
python scripts/random_agent.py --task Template-Ur10e-Ocs-1-v0 --num_envs 32
```

### 2. 大小写敏感

Linux 系统下路径区分大小写，务必确保安装路径与项目目录完全一致，例如：

```bash
source/UR10e_OCS_1
```

不要写成其他大小写形式。

### 3. 后续添加接触网（OCS）模型

后续若要加入接触网模型，可在 `SceneCfg` 中继续添加 `AssetBaseCfg`，并将其指向你的 `.usd` 文件。例如：

```python
ocs = AssetBaseCfg(
    prim_path="{ENV_REGEX_NS}/OCS",
    spawn=sim_utils.UsdFileCfg(
        usd_path="/你的/OCS模型路径/ocs.usd"
    )
)
```

## 完整流程总结

你需要执行的完整流程如下：

### 1. 新建项目

```bash
cd ~/Documents/IsaacLab
./isaaclab.sh --new
```

### 2. 进入项目并安装

```bash
cd /home/isst/Action/04lab_example/UR10e_OCS_1
conda activate env_isaaclab
python -m pip install -e source/UR10e_OCS_1
```

### 3. 修改配置文件

编辑：

```bash
source/UR10e_OCS_1/config/ur10e_catenary_inspection/env_cfg.py
```

完成以下修改：

- 将机器人导入替换为 `UR10_CFG`
- 将 `SceneCfg` 中的机器人替换为 UR10e
- 将观测项简化为 `joint_pos` 和 `joint_vel`
- 将奖励项简化为 `alive` 和 `action_rate`

### 4. 验证任务是否成功注册

```bash
python scripts/list_envs.py
```

### 5. 启动随机策略测试

```bash
python scripts/random_agent.py --task Template-Ur10e-Ocs-1-v0
```
